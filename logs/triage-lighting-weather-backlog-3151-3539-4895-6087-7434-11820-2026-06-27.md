## Decisions

### Darker Nights

Keep `modDarkerNights`.

Skip the old `3151` Darker Nights compatibility patch. It carries broad script replacements for `ingameMenu.ws`, `r4Player.ws`, `r4Game.ws`, `environmentManager.ws`, `playerWitcher.ws`, and meditation/unconscious state files. Those are hot files in this stack and should not be replaced by an older patch.

### Atmospheric Nights / 6087

Skip `6087`.

Inspection showed the archive is only a tiny `modAtmosphericNights` payload:

- `modAtmosphericNights\content\blob0.bundle`
- `modAtmosphericNights\content\metadata.store`

`modAtmosphericNights` was already archived because it was shadowed by `modDarkerNights`. Reintroducing this payload would only reopen a redundant night-visual lane.

### True Fires

Keep `modTrueFires` and `modATrueFiresHDReworkedProjectCompatibility`.

The HDRP compatibility patch is required for the current HDRP/True Fires pairing. Natural Torchlight may look redundant from asset ownership, but its script/menu behavior was manually grafted into the active merged stack, so it remains intentional.

### Benis Toussaint Lighting

Keep the low-priority Benis install for now.

It contributes unique Toussaint/Blood and Wine lighting content while losing exact conflicts to the current Filterless Toussaint / Realistic Weather winners. If Toussaint looks wrong later, choose a Toussaint visual owner deliberately rather than removing pieces blindly.
